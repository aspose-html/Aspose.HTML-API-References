## AddEventListener(string, DOMEventHandler, bool) {#addeventlistener}

此方法允许在事件目标上注册事件侦听器。

```csharp
public void AddEventListener(string type, DOMEventHandler handler, bool useCapture)
```

| 范围 | 类型 | 描述 |
| --- | --- | --- |
| type | String | 用户注册的事件类型 |
| handler | DOMEventHandler | 在事件发生时调用[`DOMEventHandler`](../../../aspose.html.dom.events/domeventhandler)。 |
| useCapture | Boolean | 如果为真，useCapture 表示用户希望启动捕获。 启动捕获后，所有指定类型的事件将被分派到注册的 [`IEventListener`](../../../aspose.html.dom.events/ieventlistener) 在被分派到树中它们下方的任何事件目标之前。 通过树向上冒泡的事件不会触发[`IEventListener`](../../../aspose.html.dom.events/ieventlistener)指定使用捕获。 |

### 评论

如果[`IEventListener`](../../../aspose.html.dom.events/ieventlistener)被添加到[`EventTarget`](../../eventtarget)在处理事件时，它不会被当前操作触发 但可能会被触发在事件流的后期阶段，例如冒泡阶段。

如果在同一个EventTarget使用相同的参数，重复的实例被丢弃。 它们不会导致[`IEventListener`](../../../aspose.html.dom.events/ieventlistener)被调用两次，并且由于它们被丢弃，因此不需要使用 [`RemoveEventListener`](../removeeventlistener) 方法。
